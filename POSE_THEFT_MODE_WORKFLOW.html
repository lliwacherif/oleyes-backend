Advanced Theft Mode (Pose AI) — Workflow

What happens when the toggle is ON and the user clicks EXECUTE

1
User Action

Toggle ON in Settings

User flips "Advanced Theft Mode (Pose AI)" to ON. The frontend saves poseTheftMode = true to localStorage. This persists across page reloads.

2
Frontend

EXECUTE Button → API Call

User clicks EXECUTE on a camera. The frontend reads the setting from context and sends the POST request with pose_theft_mode: true in the JSON body alongside rtsp_url, camera_id, etc.

3
Backend — Model Switch

Load YOLO-Pose Instead of Standard YOLO

The backend sees pose_theft_mode = True and loads yolo26n-pose.pt (skeleton model) instead of the standard yolo26n.pt (bounding-box only). Both models stay in memory — no reload needed on the next job.

4
Backend — Frame Processing

Extract Bounding Boxes + 17 Skeleton Keypoints

Each video frame goes through the pose model. The backend extracts the usual bounding boxes plus 17 COCO keypoints per person (nose, eyes, shoulders, elbows, wrists, hips, knees, ankles). These keypoints are passed into the Logic Engine.

5
Logic Engine — Kinematic Analysis

Detect Hand-to-Item and Hand-to-Pocket

The Logic Engine runs two geometric checks every frame:

1. Wrist → Item: Is either wrist (keypoints 9, 10) overlapping or touching a stealable object's bounding box (handbag, phone, laptop, etc.)?
If yes → log POSE_KINEMATIC: Person#3 wrist interacted with Handbag#7

2. Wrist → Hip: Is either wrist close to either hip (keypoints 11, 12) on the same person?
If yes → log POSE_KINEMATIC: Person#3 wrist moved to hip/pocket area

6
LLM Analysis — Pre-Crime Sequence

Specialized Theft Prompt Sent to LLM

The scene text (with kinematic events) is batched and sent to the LLM with a specialized prompt that replaces the standard theft rule:

"ADVANCED POSE THEFT MODE: Look for the Pre-Crime Sequence —
  1. Browsing (wrist interacts with item)
  2. Concealment (wrist moves to hip/pocket/backpack)
  3. Flight (item disappears, person speeds up or leaves)
If you see Concealment followed by the item disappearing, flag as HIGH RISK THEFT immediately."

7
SSE Stream → Dashboard

Results Streamed Back to the UI

The LLM responds with a risk score, label, and explanation. This is streamed to the frontend via the same SSE connection in the same JSON format as standard mode. The dashboard displays the risk level, alerts, and zone intrusions as usual — no UI parsing changes needed.

Standard Mode (Toggle OFF)

  • Uses yolo26n.pt (bounding boxes only)
  • Proximity = box-to-box distance
  • Theft = item disappears near person
  • No skeleton or keypoint data
  • Standard temporal theft prompt to LLM

Pose Theft Mode (Toggle ON)

  • Uses yolo26n-pose.pt (boxes + 17 keypoints)
  • Wrist-to-item geometric intersection
  • Wrist-to-hip concealment detection
  • Pre-crime sequence: Browse → Conceal → Flight
  • Specialized kinematic theft prompt to LLM
